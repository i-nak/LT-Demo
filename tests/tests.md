# LTDemo 動作テスト手順

> このファイルは **Xcode 27 の Intelligence チャット（Claude）に読ませて**使う。
> シミュレータを操作して、下のテストケースを上から順に実行してもらう。

## エージェントへの指示

- LTDemo をシミュレータで起動した状態から始める。
- 各ケースを**上から順に**実行する。1ケースごとに**一覧画面に戻してから**開始する（前のケースの状態を残さない）。
- 「操作」をシミュレータ上で実際に行い、「期待」を満たすか**画面を見て**判定する。
- 判定は **OK / NG / 判定不能** の3値。曖昧なら推測で OK にせず「判定不能」にする。
- NG のときは、何がどう問題か（例：本文が枠下端から何 pt はみ出し選択肢に重なる）を**根拠つき**で書く。
- 最後に結果を**表（ケース / 判定 / 理由）**でまとめて報告する。

## テストケース

### 1. 最長問題の見切れ確認
- 操作: 一覧で「問題5（最長問題）」をタップして詳細を開く
- 期待: 問題文がカード（枠）内に収まり、下の選択肢ボタンに重なって見切れていないこと

### 2. 短い問題の正常表示
- 操作: 一覧で「問題1」をタップして詳細を開く
- 期待: 問題文が枠内に収まり、選択肢と重ならず正常に表示されること

### 3. 一覧 → 詳細 → 戻る
- 操作: 任意の問題を開き、ナビゲーションバーの戻るボタンで一覧へ戻る
- 期待: 詳細画面が開き、戻ると問題一覧に戻れること

### 4. 全問ざっと表示崩れチェック
- 操作: 問題1〜5 を順に開き、それぞれ画面を確認する
- 期待: いずれの問題でも、テキストの見切れ・要素の重なり・明らかなレイアウト崩れが無いこと
